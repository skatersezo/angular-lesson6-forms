Template driven forms

In the template driven approach, everything you do, you do it on the view.

ngModel is not only for two-way data binging, it is a directive from forms module which adds control functionality in the forms.

Validation with TD approach is made by directives. You can find out their names, by searching for "validator" in the official docs: https://angular.io/api?type=directive - everything marked with "D" is a directive and can be added to your template. Additionally, you might also want to enable HTML5 validation (by default, Angular disables it). You can do so by adding the ngNativeValidate to a control in your template.

ngModel can be used as a simple form control with no binding, one way binding for giving a default value, and two way binding to instantly put the value.

Reactive forms

In the reactive approach, you synchronize the form in the view, but you configure it in the logic.

With the reactive approach you define a FormGroup object in the *.ts file that will hold inside all the FormControls.

Instatiating the FormGroup object allows you to pass a Json object with the properties named as the formControlName directive string you provided in the template.

Each FormControl that you instantiate in the FormGroup accepts 2 arguments, the default value and the validator or array of validators. The validations are done by the Validators object provided by Angular.

Within the reactive approach you're allowed to create nested FormGroups.

Besides FormControls it is posible also to instantiate a FormArray inside the FormGroup that will contain as many FormControls as we want. It is useful to provide dinamically FormControls to an existing FormGroup.

Validators are functions, and Angular provides a lot out of the box, but it is also possible to create custom validators by writting a function that will receive a FormControl and will return a key-value pair object.

If you need a Validation from a server, it is possible to create Async validators. But it is important to remember, this async validators are provided as a third argument to the FormControls.
